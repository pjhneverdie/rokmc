# JSP가 나온 이유
jsp 자체가 서블릿 방식의 짬뽕을 조금이라도 분리하려는 시도.

서블릿: PrintWriter로 HTML을 한 줄씩 출력해야 함 → 코드 지저분

JSP: HTML 문서 안에 필요한 부분만 자바 코드를 넣으면 됨 → 훨씬 읽기 편함

h1 <%= new java.util.Date() %> h1 이런 느낌

CSS/JS 통합 관리 용이

JSP는 기본적으로 HTML 문서처럼 작성하므로, <link>로 CSS 연결하고 <script>로 JS 연결하는 것이 서블릿보다 훨씬 직관적임

즉, 서블릿에서 데이터 처리 후 JSP로 넘겨 화면 표시 → 코드 관리가 깔끔해짐
